# Révisé et rapport

1. **As-tu été au bout de ta mission ?**
   - Si oui → passe à l’étape 2.
   - Si non → poursuis le travail, puis reviens à l’étape 1.

2. **Réviser**
   - Spawner un sous-agent QA.
   - Lui fournir tout le nécessaire pour produire un rapport QA dans les règles de l’art :
     - le lien vers le plan (chemin complet, ex. `C:\Users\Strophe\.cursor\plans\nom-du-plan.md`) s’il y en a un ;
     - les fichiers à auditer et toutes les références utiles.
   - Demander au sous-agent : un **rapport QA** sur tout ça, court et précis, sans rien négliger, **sans créer de fichier** — uniquement te le restituer en fin de run dans le chat.

3. **Appliquer les révisions**
   - À partir des résultats QA, exécuter les corrections ou ajustements identifiés.

4. **lister les dettes**
   - faire la liste de tout ce qui n'a pas été traité ou est resté au niveau secondaire, les petites dettes, les micro choses à faire plus tard

5. **rembourser ces dettes**
   - executer toutes celles qui sont possible d'etre faites. Privilegier un subadgent si la liste est conséquente.

6. **Rédiger le rapport au planner**
   - Écrire ton rapport final ici dans le chat, dans une **fenêtre de code MD** prête à copier-coller (format attendu : les points majeurs headlines + Verdict exécutable, les ref des artefacts crées).
