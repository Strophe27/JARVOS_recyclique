# Backlog 1.4.5 - Consolidation

## Cap

La base active de travail est `recyclique-1.4.4/`.

La cible `1.4.5` est une consolidation de l'existant :
- stabilite,
- securite,
- hygiene technique,
- parite terrain prioritaire,
- documentation et pilotage clairs.

La refonte JARVOS precedente reste une archive secondaire. Elle peut servir de reference ponctuelle, mais elle ne pilote plus le produit.

## Hors scope immediat

Pour ce cycle, on ne relance pas :
- la refonte Vite/FastAPI comme cible principale,
- la modularisation lourde,
- le SSO,
- la caisse hors ligne,
- les extension points de type `LayoutConfigService` / `VisualProvider`,
- les nouvelles couches techniques non necessaires a la consolidation.

## Premier lot recommande

Le premier lot concret recommande pour lancer `1.4.5` est :

1. unifier la version produit affichee,
2. nettoyer le bruit debug evident,
3. stabiliser les tests backend documentes comme fragiles,
4. preparer ensuite un lot caisse/import legacy si la base de verification est redevenue fiable.
