## 3. Difficultés et frottements (sans blâme — analyse système)

**Elena (Dev) :** « La charge cognitive est élevée : beaucoup de fichiers `references/`, index à tenir à jour, risque de désalignement sprint / en-têtes de story si on ne synchronise pas en fin de Story Runner. »

**Charlie (Dev senior) :** « L'inventaire ancien repo vs snapshot OpenAPI “live” — il faut des repères clairs pour ne pas mélanger les versions quand on cite des endpoints. »

**Dana (QA) :** « Les tests “contrat markdown” peuvent diverger du wording réel de l'artefact ; la réponse n'est pas de forcer le markdown, c'est d'ajuster les assertions — mais il faut le décider explicitement. »

**Alice (Product Owner) :** « **HITL** : les checks automatisés ne remplacent pas la validation métier sur les signaux bandeau et la matrice Paheko — c'est assumé dans les stories, mais il faut le rappeler à chaque jalon. »

**Motifs transverses :**

- **Doc lourde** : plusieurs artefacts datés ; l'index `references/artefacts/index.md` est indispensable.
- **Cohérence nommage fichiers** (ex. JSON **08** OpenAPI figée) : une ambiguïté corrigée tôt évite la confusion en relecture.
- **Epic “doc”** : la “vélocité” se mesure en complétude et traçabilité, pas en lignes de code — le `sprint-status` reflète bien **done** quand les critères sont tenus.

## 7. Actions et engagements (SMART — sans estimation temporelle chiffrée)

**Processus**

1. **Synchroniser statuts** en fin de chaque story : `sprint-status.yaml` + en-tête fichier story — **responsable :** équipe / Story Runner — **critère de succès :** aucune story **done** dans le YAML encore en **review** dans le fichier.
2. **Rappel HITL** sur les jalons doc critiques (matrice, signaux) — **responsable :** Product / Strophe — **critère :** checklist validée ou écart explicite consigné.

3. **Garder une seule source OpenAPI reviewable** ; toute évolution backend qui touche l'API passe par mise à jour contrôlée de `contracts/openapi/recyclique-api.yaml` — **responsable :** Dev Epic 2 — **critère :** PR ou changement traçable lié aux stories **2.x**.
4. **Référence JSON figée** (`references/artefacts/2026-04-02_08_openapi-recyclique-live-recyclic-local.json`, alias « artefact 08 ») : usage **lecture seule** pour alignement — ne pas la confondre avec `contracts/openapi/recyclique-api.yaml` (YAML reviewable, évolutif) — **responsable :** équipe contrats — **critère :** mentions dans README / index à jour.

**Préparation Epic 2**

5. **Prioriser** les premières stories socle (session, `ContextEnvelope`) selon `guide-pilotage-v2.md` — **responsable :** PO + architecte — **critère :** ordre des stories **2.1+** validé sans dépendance circulaire avec les livrables 1.x.
