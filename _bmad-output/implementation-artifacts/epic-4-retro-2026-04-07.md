## 5. Lecons retenues

1. **Un gate produit doit expliciter le lieu de preuve.** Si la validation attend l'application servie, ce critere doit etre visible tres tot dans les stories.
2. **La preuve automatisee et la preuve humaine doivent etre separees puis reunies.** L'une donne la reproductibilite, l'autre ferme le doute produit.
3. **Le dernier kilometre runtime doit etre une responsabilite explicite.** Charger les bons manifests dans le bundle servi est une tache a part entiere.
4. **Le fallback visible est un livrable produit, pas juste technique.** Le 401 avant login a servi de preuve de posture defensive, pas d'anomalie.

---

## 6. Preparation Epic 5

**Titre suivant (`epics.md`) :** Epic 5 — shell, dashboard et admin transverses dans `Peintre_nano`.
